# User Guides & Tutorials Skill

## Quick Start

### User Guide Structure

```markdown
# Getting Started with [Product]

## Overview
Brief (1-2 paragraph) explanation of what the product does and who it's for.

## System Requirements
- Minimum specifications
- Supported platforms
- Dependencies

## Installation

### Step 1: Download
Instructions with screenshots/links

### Step 2: Configure
Configuration steps with examples

### Step 3: Verify
How to confirm successful setup

## Basic Usage

### Your First Task
Step-by-step guide with:
- Clear numbered steps
- Screenshots at key points
- Expected results

### Common Actions
Table of frequent tasks with links

## Advanced Features

### Feature 1: Detail with Examples
Detailed explanation with code or configuration examples

## Troubleshooting

### Issue: [Common Problem]
**Symptom**: How to recognize this issue
**Cause**: Why it happens
**Solution**: Step-by-step fix

## FAQ

**Q: Common question?**
A: Clear, concise answer

## Next Steps
- Link to advanced tutorials
- Related features to explore
- Community resources

## Support
- Contact information
- Documentation links
- Community forums
```

## Best Practices

### Writing for Users

✅ **DO:**
- Use simple, clear language
- Focus on what users want to do
- Include plenty of examples
- Provide screenshots for visual steps
- Use numbered lists for steps
- Include expected results
- Link related content
- Update regularly

❌ **DON'T:**
- Use technical jargon without explanation
- Write long paragraphs
- Assume user knowledge
- Skip error handling
- Use outdated screenshots
- Create dead links
- Overwhelm with options

### Structure Guidelines

**Heading Hierarchy:**
- H1: Guide title
- H2: Major sections
- H3: Subsections
- H4: Details

**Information Density:**
- One concept per section
- 3-4 paragraphs max per section
- Use lists to break up text
- Include visual breaks

### Screenshots & Visuals

1. **Use high-quality screenshots**
2. **Add annotations/arrows** pointing to relevant areas
3. **Highlight fields** users need to fill
4. **Show both desktop and mobile** if applicable
5. **Update regularly** when UI changes

## Common Guide Patterns

### Conceptual Guide
Explains "what is" and "why" without task focus

### Task Guide
Step-by-step "how to" accomplish specific goal

### Reference Guide
Complete list of options, parameters, features

### Tutorial
Learning-focused, teaches concepts through practice

### API Guide
Technical reference for developers

## Tools for Creating Guides

### Documentation Platforms
- Markdown-based: Notion, Coda, GitBook
- WYSIWYG: Confluence, Google Docs
- Static: MkDocs, Docusaurus, Hugo

### Screenshot Tools
- Snagit, ShareX, Loom
- Browser extensions: Nimbus, Awesome Screenshot
- Built-in: macOS Screenshot, Windows Snip & Sketch

### Formatting
- Markdown for simplicity
- HTML for control
- ReStructuredText for Sphinx
